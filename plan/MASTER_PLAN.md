# Master Plan

This folder contains canonical lab briefs used to drive implementation order.

## Lab Sequence
1. `plan/lab_01_foundations.md` — core runtime/config/test foundation.
2. `plan/lab_02_navigation.md` — navigation planning and execution scaffold.
3. `plan/lab_03_perception.md` — perception pipeline scaffold.
4. `plan/lab_04_manipulation.md` — manipulation/control (upcoming).
5. `plan/lab_05_integration.md` — full-system integration/evaluation (upcoming).

## Process Rule
For each lab, follow:
1. Create lab folder structure.
2. Write `tasks/PLAN.md`.
3. Write `tasks/ARCHITECTURE.md`.
4. Generate/update `tasks/TODO.md` after each completed step.
5. Log blockers and lessons in `tasks/LESSONS.md`.
6. Implement code + tests + scripts + docs.
7. Update root `README.md` when lab reaches completion.
